# 4\. 技术要求与限制

## 4.1 允许使用的技术

* \[ ] HTML5
* \[ ] CSS3
* \[ ] 原生 JavaScript
* \[ ] CSS Grid
* \[ ] Flexbox
* \[ ] CSS 自定义属性
* \[ ] CSS 媒体查询
* \[ ] 本地字体文件
* \[ ] 本地图片、图标和静音视频

## 4.2 禁止或不采用的技术

* \[ ] 不使用 React、Vue、Angular、Next.js 等前端框架。
* \[ ] 不使用 Tailwind、Bootstrap 等大型样式框架。
* \[ ] 不使用 npm、Node.js、Webpack、Vite 等构建流程。
* \[ ] 不使用数据库、登录、注册或后台管理系统。
* \[ ] 不使用 WebGL、Three.js 或复杂三维效果。
* \[ ] 不使用滚动劫持。
* \[ ] 不依赖外部 CDN 才能显示核心内容。
* \[ ] 不使用 iframe 嵌入参考网站。
* \[ ] 不复制参考网站源代码。
* \[ ] 不直接使用参考网站的图片、标志、口号和商业内容。
* \[ ] 不使用绝对本地路径，例如 `C:\\Users\\...`。
* \[ ] 不使用必须连接网络才能工作的核心功能。

## 4.3 本地运行要求

* \[ ] 网站可以由 Dreamweaver 建立为本地站点。
* \[ ] 网站主要页面可以在浏览器中直接打开。
* \[ ] 所有内部链接使用文档相对路径。
* \[ ] 图片、字体、CSS 和 JavaScript 均保存在项目文件夹内部。
* \[ ] 不启动服务器时，首页、分类页和详情页仍可正常浏览。
* \[ ] JavaScript 失效时，主要文字、图片和项目卡片仍然可见。

## 4.4 文件命名要求

* \[ ] 文件名只使用小写英文字母、数字和连字符。
* \[ ] 文件名不使用空格。
* \[ ] 文件名不使用中文。
* \[ ] 文件名不使用括号、井号和其他特殊符号。
* \[ ] 页面扩展名统一为 `.html`。
* \[ ] 样式文件扩展名统一为 `.css`。
* \[ ] 脚本文件扩展名统一为 `.js`。
* \[ ] 网页图片优先使用 `.webp`、`.jpg` 或 `.png`。

本项目最低完成 **20 个独立 HTML 页面**。

|页面类型|最低数量|说明|
|-|-:|-|
|首页|1|项目入口与精选视觉实验|

# 7\. 视觉要求

## 7.1 总体视觉方向

网站应同时具有：

* 彩色模块化构成
* 黑色粗边框
* 超大标题
* 不对称拼贴
* 档案式网格
* 清晰元数据
* 项目编号系统
* 克制的详情页
* 视觉实验与资料目录并存的感觉

## 7.2 基础色

* Ink：`#111111`
* Paper：`#F2EFE7`
* Soft Grey：`#DADADA`

## 7.3 分类色

* TYPE：`#DFFF42`
* IMAGE：`#FF4933`
* SYSTEM：`#3155FF`
* FIELD NOTES：`#7958FF`

## 7.4 视觉限制

* \[ ] 卡片默认不使用圆角。
* \[ ] 不使用玻璃拟态。
* \[ ] 不使用霓虹发光。
* \[ ] 不使用大面积渐变。
* \[ ] 不使用装饰性阴影。
* \[ ] 不随机改变分类颜色。
* \[ ] 不横向拉伸图片。
* \[ ] 不把所有页面做成同样密集的彩色拼贴。
* \[ ] 详情页以阅读和展示图片为主，视觉应比首页克制。

## 7.5 字体要求

网站使用三类字体：

* Display：用于超大英文标题和分类标题
* Text：用于中文正文和项目说明
* Metadata：用于编号、日期、媒介和标签

字体文件应保存在：

```text
assets/fonts/
```

核心文字不得依赖联网字体服务。

## 8.1 全站基础功能

每个分类页必须包含：

* \[ ] 分类名称
* \[ ] 分类中文名称
* \[ ] 分类简短定义
* \[ ] 3 个对应实验
* \[ ] 返回 Archive 的链接
* \[ ] 对应分类色
* \[ ] 与 Archive 一致的卡片规则

## 8.4 详情页必须功能

每个详情页必须包含：

* \[ ] 项目编号

* \[ ] 分类
* \[ ] 日期
* \[ ] 媒介
* \[ ] 状态

* \[ ] 主视觉
* \[ ] 项目简介
* \[ ] 内容图或过程图
* \[ ] 观察笔记或结果说明
* \[ ] 返回分类
* \[ ] 返回 Archive
* \[ ] 上一篇
* \[ ] 下一篇

## 8.5 移动端必须功能

* \[ ] 移动端导航可以打开和关闭。
* \[ ] 菜单按钮具有文字或可理解的无障碍名称。
* \[ ] 菜单状态使用 `aria-expanded`。
* \[ ] 点击链接后菜单自动关闭。
* \[ ] 按 Escape 可以关闭展开的菜单。
* \[ ] 筛选按钮在窄屏下可以换行或横向滚动。

## 8.6 动效要求

必须包含轻量动效：

* \[ ] 卡片悬停时图片轻微放大
* \[ ] 卡片元数据出现或移动
* \[ ] 页面内容轻微淡入
* \[ ] 视图切换具有短暂过渡
* \[ ] 移动菜单具有短暂开合过渡

动效限制：

* 动画时长建议为 150—300 毫秒。
* 主要使用 `opacity` 和 `transform`。
* 动画不得阻止点击。
* 页面载入时不得长时间隐藏正文。
* 必须支持 `prefers-reduced-motion`。
* 减少动态模式下应关闭非必要移动效果。

## 8.7 不要求的功能

本项目不要求：

* 表单提交后台
* 用户注册
* 用户登录
* 搜索数据库
* 购物车
* 支付
* 评论
* 文件上传
* 在线管理后台
* 多语言自动切换
* 地图
* 实时聊天
* 复杂声音交互

联系页可以使用静态邮箱链接，不需要真正的后台表单。

# 9\. 屏幕适配要求

## 9.1 必须检查的屏幕宽度

* \[ ] 375 px：手机
* \[ ] 768 px：平板竖屏
* \[ ] 1024 px：平板横屏或小型电脑
* \[ ] 1440 px：常见桌面显示器

## 9.2 桌面端要求

* \[ ] 首页采用左右分栏或模块拼贴结构。
* \[ ] Archive 使用 3—4 列网格。
* \[ ] 详情页主视觉和元数据可以采用双栏。
* \[ ] 页面正文不应横跨整个屏幕。
* \[ ] 超大标题不应被裁切。
* \[ ] 导航和工具栏清晰可见。

## 9.3 平板端要求

* \[ ] 首页分栏比例可以缩小或改为上下排列。
* \[ ] Archive 改为 2—3 列。
* \[ ] 工具栏允许换行。
* \[ ] 详情页可保留双栏或改为单栏。
* \[ ] 图片保持原始比例。

## 9.4 手机端要求

* \[ ] 页面改为单列阅读顺序。
* \[ ] 首页固定侧栏取消固定。
* \[ ] Archive 主要采用单列或双列。
* \[ ] 详情页采用单列。
* \[ ] 导航改为可开合菜单。
* \[ ] 正文不小于可读尺寸。
* \[ ] 按钮具有足够点击面积。
* \[ ] 页面不存在意外横向滚动。
* \[ ] 表格或工具栏不会超出屏幕。
* \[ ] 图片、视频不超出容器。

## 9.5 响应式验收标准

* \[ ] 375 px 下无明显横向溢出。
* \[ ] 768 px 下卡片排列正常。
* \[ ] 1024 px 下导航和详情页结构正常。
* \[ ] 1440 px 下页面不会过度稀疏。
* \[ ] 所有图片不变形。
* \[ ] 中文与英文标题不异常重叠。
* \[ ] 页面关键按钮不会被遮挡。

# 10\. 可访问性要求

* \[ ] 每个页面设置正确的 `lang` 属性。
* \[ ] 每个页面具有唯一的 `<title>`。
* \[ ] 每个页面只有一个主要 `<h1>`。
* \[ ] 标题层级连续。
* \[ ] 信息图片具有准确的 `alt`。
* \[ ] 装饰图片使用空 `alt`。
* \[ ] 所有按钮可通过键盘操作。
* \[ ] 所有链接可通过键盘访问。
* \[ ] `focus-visible` 状态清晰。
* \[ ] 分类不能只依赖颜色区分。
* \[ ] 按钮文字能够说明功能。
* \[ ] GRID、LIST、INDEX、MIX 使用真实 `<button>`。
* \[ ] 当前筛选或视图按钮使用 `aria-pressed`。
* \[ ] 移动菜单使用 `aria-expanded`。
* \[ ] 页面支持 `prefers-reduced-motion`。
* \[ ] JavaScript 关闭后主要内容仍然可见。
