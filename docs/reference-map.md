# PERSONAL VISUAL ARCHIVE
# 参考资料拆解与使用说明

## 1. 文件用途

本文件说明 `references/` 中每张本地参考截图：

- 实际展示什么；
- 可以学习什么；
- 用于本项目的哪个部分；
- 禁止复制什么。

参考截图只提供视觉方法和信息组织方法，不直接决定页面内容，也不属于网站正式素材。

不得把 `references/` 中的截图放入正式网页，不得复制参考网站的图片、文案、标志、项目内容、源代码或完整页面构图。

---

## 2. 项目规则优先级

发生冲突时，按以下顺序执行：

1. `docs/course-requirements.md`
2. `docs/sitemap.md`
3. `wireframes/`
4. `docs/design-system.md`
5. `docs/reference-map.md`
6. `references/` 中的截图

具体职责：

- 页面名称和页面关系以 `docs/sitemap.md` 为准。
- 页面布局以 `wireframes/` 为准。
- 色彩、字体、边框和间距以 `docs/design-system.md` 为准。
- 参考截图只用于学习视觉节奏和信息组织。
- 参考网站不能改变 A01—A12 的项目内容或三级页面结构。

## 5. 本项目自行规划的功能

以下功能属于 PERSONAL VISUAL ARCHIVE 自行规划，不来源于上述参考网站当前页面：

### 分类筛选

- ALL
- TYPE
- IMAGE
- SYSTEM
- FIELD NOTES

### 视图切换

- GRID
- LIST

### 排序方式

- INDEX
- MIX

功能含义：

- GRID：以图片卡片展示项目。
- LIST：以编号、标题和元数据组成列表。
- INDEX：恢复 A01—A12 的固定编号顺序。
- MIX：随机改变当前项目卡片顺序。

正确表述：

> 本项目借鉴 Tristan Bagot 的项目列表和元数据组织方法，并自行规划分类筛选、GRID / LIST 与 INDEX / MIX。

不得写成：

- “复制 Tristan Bagot 的筛选器”；
- “复刻 Tristan Bagot 的随机排列”；
- “按照 Tristan Bagot 的 GRID / LIST 功能制作”。

- 截图无法确认的信息必须标为“不确定”，不得虚构。
- 不得下载参考网站素材。
- 不得复制参考网站源代码。
- 不得把参考截图、截图文字或原项目内容放入正式网页。
- 不得完全照搬截图的完整构图。
- 不得因参考网站存在某类页面而擅自新增页面。
- 页面名称和关系必须服从 `docs/sitemap.md`。
- 页面布局必须服从 `wireframes/`。
- 视觉规则必须服从 `docs/design-system.md`。
