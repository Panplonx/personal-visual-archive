# PERSONAL VISUAL ARCHIVE
# 页面布局与交互规范

## 1. 总体规则

- 使用 HTML5、CSS3 和原生 JavaScript。
- 全站支持 375px、768px、1024px 和 1440px。
- 页面使用统一主导航：
  HOME / ARCHIVE / INDEX / ABOUT / CONTACT。
- 桌面端允许使用多栏布局。
- 手机端全部转换为正常单列阅读。
- 手机端不得依赖 hover 才能获得关键信息。
- 正式内容来自 content、archive-manifest 和 assets/images。
- 不得根据参考截图虚构项目、分类和文案。

---

## 2. 首页桌面端

文件：`index.html`

结构：

1. 顶部粘性导航。
2. 左侧档案控制台，约占 36%。
3. 右侧实验拼贴网格，约占 64%。
4. 左侧控制台在桌面端使用 sticky。
5. 整个页面正常纵向滚动。
6. 左侧包含：
   - 网站英文标题
   - 中文标题
   - 项目简介
   - STATUS
   - TOTAL ENTRIES: 012
   - CATEGORIES: 04
   - ENTER ARCHIVE
   - VIEW INDEX
   - ABOUT
7. 右侧使用 A01、A04、A07、A10 等真实项目组成拼贴。
8. 卡片悬停显示编号、标题和分类。
9. 点击卡片进入对应详情页。
10. 全部内容结束后显示页脚。

---

## 3. 首页手机端

文件：`index.html`

结构：

1. 顶部显示站点名称和菜单按钮。
2. 点击菜单按钮展开主导航。
3. 左侧控制台取消 sticky。
4. 页面顺序为：
   - 标题
   - 简介
   - 状态信息
   - ENTER ARCHIVE
   - 项目卡片
   - 页脚
5. 项目卡片使用单列或最多双列。
6. 重要元数据默认可见，不依赖 hover。

---

## 4. Archive 网格状态

文件：`archive.html`

必须包含：

1. 页面标题和简介。
2. 分类筛选：
   - ALL
   - TYPE
   - IMAGE
   - SYSTEM
   - FIELD NOTES
3. 查看方式：
   - GRID
   - LIST
4. 排列方式：
   - INDEX
   - MIX
5. GRID 状态显示 12 张项目卡片。
6. 每张卡片使用真实封面、编号、标题和分类。
7. 点击进入对应详情页。
8. 页面底部包含页脚。

---

## 5. Archive 列表状态

文件：`archive.html`

要求：

1. 与 GRID 共用同一页面和工具栏。
2. LIST 状态显示 12 行项目。
3. 每行包含：
   - 编号
   - 英文标题
   - 中文标题
   - 分类
   - 年份
4. 桌面端悬停可显示封面预览。
5. 手机端隐藏非必要字段，不依赖悬停。
6. 分类按钮在小屏幕下换行或横向滚动。

---

## 6. 分类页

文件：

- `collections/type.html`
- `collections/image.html`
- `collections/system.html`
- `collections/field-notes.html`

共同结构：

1. 返回 Archive。
2. 超大分类标题。
3. 分类中文名称和说明。
4. 分类元数据。
5. 3 个所属项目。
6. 项目卡片包含封面、编号、标题和摘要。
7. 点击进入详情页。
8. 页面底部再次提供返回 Archive。

---

## 7. 详情页

文件：`entries/a01...a12`

共同结构：

1. 主导航。
2. 面包屑或返回分类链接。
3. 标题和元数据。
4. 一张主图。
5. 80—200 字中文项目说明。
6. 至少 3 张详情图或过程图。
7. 图片说明。
8. 观察笔记。
9. 返回分类。
10. 返回 Archive。
11. 上一篇。
12. 下一篇。

桌面端首屏可以双栏，手机端改为单列。

---

## 8. Index 页面

文件：`index-page.html`

必须包含：

1. 页面标题和简短说明。
2. A01—A12 共 12 行文本索引。
3. 每行包含编号、标题、分类和年份。
4. 桌面端右侧提供悬停封面预览。
5. 手机端取消悬停预览。
6. 点击条目进入对应详情页。

---

## 9. About 页面

文件：`about.html`

必须包含：

1. 项目总说明。
2. TYPE、IMAGE、SYSTEM、FIELD NOTES 四类定义。
3. 制作方式和工具。
4. 作者或课程信息。
5. 进入 Archive 和 Contact 的链接。
6. 不得虚构姓名、班级、奖项或商业客户。

---

## 10. Contact 页面

文件：`contact.html`

结构：

1. 页面标题。
2. 个人页（超链接文字）
3. 项目和课程说明。
4. 返回 About。
5. 返回首页。

---

## 11. 404 页面

文件：`404.html`

必须包含：

1. 错误提示。
2. 返回首页。
3. 返回 Archive。
